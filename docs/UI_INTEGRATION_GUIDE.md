# API Integration Guide for UI

## 🎯 Endpoint to Call

**POST** `/generate-mcqs`

**Base URL**: `https://your-deployed-app.onrender.com`

---

## 📤 Request Format

### Headers
```
Content-Type: application/json
```

Optional (if API key is configured):
```
X-API-Key: your-api-key-here
```

### Request Body (JSON)

```json
{
  "category": "category_name_here"
}
```

---

## ✅ Valid Categories

The UI should send one of these category names in the request:

1. `circular_economy_and_waste_reduction`
2. `sustainability_strategy_and_compliance`
3. `sustainable_agriculture_and_natural_resources`

## ⚠️ Error Responses

### Invalid Category (400 Bad Request)
```json
{
  "detail": "Category must be one of: ['circular_economy_and_waste_reduction', 'sustainability_strategy_and_compliance', 'sustainable_agriculture_and_natural_resources']"
}
```

### Missing Category (422 Unprocessable Entity)
```json
{
  "detail": [
    {
      "loc": ["body", "category"],
      "msg": "field required",
      "type": "value_error.missing"
    }
  ]
}
```

### Server Error (500 Internal Server Error)
```json
{
  "detail": "Failed to generate MCQs for category: circular_economy_and_waste_reduction"
}
```

---

## 🔄 Response Time

**Expected:** 4-5 minutes per request

The API generates 25 questions in batches of 5, with a 61-second delay between batches to prevent rate limiting. Total time breakdown:

- Batch 1 (Q1-Q5): ~30 seconds
- Delay: 61 seconds
- Batch 2 (Q6-Q10): ~30 seconds
- Delay: 61 seconds
- Batch 3 (Q11-Q15): ~30 seconds
- Delay: 61 seconds
- Batch 4 (Q16-Q20): ~30 seconds
- Delay: 61 seconds
- Batch 5 (Q21-Q25): ~30 seconds

**Total:** ~5 minutes

**UI Recommendation:** Show a progress indicator or loading message to users.

## 💻 JavaScript/TypeScript Example

```javascript
// Generated by GitHub Copilot
async function generateMCQs(category) {
  const API_URL = 'https://your-app.onrender.com/generate-mcqs';
  
  try {
    const response = await fetch(API_URL, {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        // Include if API key is required
        // 'X-API-Key': 'your-api-key-here'
      },
      body: JSON.stringify({ category })
    });
    
    if (!response.ok) {
      const error = await response.json();
      throw new Error(error.detail || 'Failed to generate MCQs');
    }
    
    const data = await response.json();
    console.log(`Generated ${data.mcq_sets[0].total_questions} questions`);
    return data;
    
  } catch (error) {
    console.error('Error:', error.message);
    throw error;
  }
}

// Usage
generateMCQs('circular_economy_and_waste_reduction')
  .then(result => {
    console.log('MCQ Set:', result.mcq_sets[0]);
    // Display questions in UI
  })
  .catch(error => {
    // Show error message to user
  });
```

### Key Data to Extract from Response

```javascript
const response = await generateMCQs('circular_economy_and_waste_reduction');

// Access data
const category = response.category;                    // "circular_economy_and_waste_reduction"
const setNumber = response.mcq_sets[0].set_number;     // 1
const totalQuestions = response.mcq_sets[0].total_questions;  // 25
const questions = response.mcq_sets[0].questions;      // Array of 25 questions

// Each question has:
questions.forEach((q, index) => {
  console.log(`Q${index + 1}: ${q.question}`);
  console.log(`Options: A=${q.options.A}, B=${q.options.B}, C=${q.options.C}, D=${q.options.D}`);
  console.log(`Answer: ${q.correct_answer}`);
  console.log(`Explanation: ${q.explanation}`);
});
```

## 📊 Response Structure Reference

```typescript
interface MCQResponse {
  status: "success";
  category: string;
  total_sets: number;
  mcq_sets: MCQSet[];
  message: string;
}

interface MCQSet {
  category: string;
  set_number: number;
  total_questions: number;
  questions: Question[];
}

interface Question {
  question: string;
  options: {
    A: string;
    B: string;
    C: string;
    D: string;
  };
  correct_answer: "A" | "B" | "C" | "D";
  explanation: string;
}
```

## 🛡️ Security & Best Practices

1. **Timeout Handling**: Set request timeout to at least 8 minutes
2. **Loading States**: Show progress indicator during generation  
3. **Error Handling**: Display user-friendly error messages
4. **Validation**: Validate category before sending request
5. **Retry Logic**: Implement retry for network failures (not for 400/422 errors)
6. **Caching**: Consider caching responses to avoid regenerating same category
